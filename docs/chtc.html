1.1 Common CONDOR commands

  • condor_submit sub.sub
  • condor_q for quick review of queue for submitted jobs
  • condor_q -hold for quick review of held jobs
  • condor_q -af HoldReason for more detailed review of held jobs
  • condor_qedit [cluster id number] RequestMemory [memory] to increase memory. Example: condor_qedit 16892087 RequestMemory 20000. Can specific specific job with .# added to the ID. Memory is specified in MB in our training control files.
  • condor_release $USER to release all held jobs for user (e.g., after increasing memory)
  • condor_rm [jobid] or condor_rm $USER to remove a job or all jobs
  • condor_history [NetID] -limit 10 -af requestmemory memoryusage is an example of reviewing recent job history (past 10 in this example) for a subset of the history parameters. Can list all the parameters (long format) for a single job using condor_history 16892169.1 -l where the jobid is listed explicitly.

1.2 Common UNIX generic commands

  • ls -lSr to list files sorted by size (with larger sizes later). Useful to find the non-zero error files. Can have issues when applied to large numbers of files. See use of find and xargs below if needed.
  • rm to remove file(s). rm -r to remove non-empty directory
  • nano is used to edit text files. For example nano job_nums.txt to edit job_nums.txt for jobs to re-run due to errors
  • cat results_* > results_all.csv to combine all results into one file before transferring back to our server
  • chmod +x get_missing_jobs.sh to change permissions on get_missing_jobs.sh to run it. ./get_missing_jobs.sh to run it.
  • find -maxdepth 1 -name "results_*.csv" | wc -l will find results_*.csv in local folder (maxdepth 1) and pipe them into a word count that counts lines (-l)
  • cat job_nums.txt | wc -l reads contents of job_nums.text and pipes to a word count that counts lines (-l)
  • find -maxdepth 1 -name "results_*.csv" | ls -lSr gets around the too many arguments issue.
  • find -maxdepth 1 -name "results_*.csv" | xargs awk 'FNR == 1 && NR != 1 {next;}{print}' > combined_results.csv to concatenate results files without header. See tutorial on awk to understand its use for simple programming. See a tutorial on the use of xargs.

1.4 Logging In

  1. Open PuTTY. Enter Hostname: submit-1.chtc.wisc.edu, Port: 22, connection type: ssh
  2. Log in with your netid. You will have to use DUO for MFA.
  3. Open WinSCP. Use the same credentials as PuTTY. You can drag files between your computer and CHTC. You can also edit documents with this program.
